# Can an AI Board Remember the Project, Not Just the Prompt?

## Quick Answer

The next useful architecture AI workflow is not just a better prompt box.

It is a project-aware board that remembers:

- the original reference
- the first AI output
- the brief summary
- the design constraints
- the visual direction
- the next decision question

That matters because architectural work is rarely one image. It is a chain of decisions. If the AI workflow forgets that chain, every output becomes a separate experiment.

---

## The Prompt Is Not The Project

A prompt is a temporary instruction.

A project is different. A project has a brief, a client, a site, a visual direction, constraints, internal comments, rejected options, and a next step.

Most AI image workflows still treat these as separate pieces. You upload an image, write a prompt, receive an output, download it, then start again. The image may be good, but the surrounding design logic often disappears.

That creates a real problem for architects and small studios. The team has to remember:

- why this image was made
- what reference it came from
- what changed from the last version
- which constraints were important
- what should be tested next

If those things are only inside a chat thread, a filename, or someone's memory, the workflow becomes fragile.

---

## Why This Matters More Than Prompt Tricks

Prompt tricks can improve a single result.

Project memory improves the workflow around the result.

For architecture work, that difference is important. A studio does not only need a nice image. It needs a way to review the image, compare it with the brief, decide what is working, and continue the project without restarting from zero.

This is why boards still matter. Architects already use boards because they make design thinking visible. A board can hold references, material ideas, markups, options, notes, and decisions in one place.

AI work should not remove that structure. It should support it.

---

## What A Project-Aware AI Board Should Hold

A useful AI board does not need to be complicated. The first version can be simple.

It should keep four things together.

### 1. Reference Image

The reference shows where the work started. It may be a sketch, site photo, clay render, precedent, or previous option.

Without the reference, the AI output floats without context.

### 2. First Output

The output should sit near the reference so the difference is visible. This makes it easier to ask whether the AI moved the project in the right direction.

The question is not only "is this image good?"

The better question is: "does this image answer the design problem?"

### 3. Brief And Constraint Notes

The board should show the short project logic:

- what the project is trying to become
- what should not change
- what mood or material direction is being tested
- what the studio needs to decide next

These notes keep the AI work grounded.

### 4. Decision Question

Every board should end with a decision question.

For example:

- Should the next option test a brighter street edge?
- Should the material direction become warmer or more neutral?
- Should the design stay calm, or should the corner become stronger?

This turns AI output into design progress. The image becomes part of a decision, not just another file.

## What The Taipei Cowork Board Demo Is Testing

The current Nomavek Lab board direction starts with a simple Taipei Cowork Board demo.

The demo is intentionally small. It does not try to become a full project management system. It tests one practical workflow:

```text
reference image
-> first AI output
-> brief summary
-> key constraints
-> visual language
-> decision question
```

That structure is enough to show the core idea. The board is not only a place to display the result. It is a place to keep the project readable.

For small studios, this matters because many AI workflows become messy quickly. You may have ten outputs, three prompt versions, two reference folders, and no clear reason why one option is better than another.

A board gives the project a visible memory.

---

## Better AI Workflows May Look More Like Boards

The interesting shift in architecture AI is not only better image generation.

The larger shift is that AI tools are starting to become more project-aware. Instead of only generating one image from one prompt, the workflow can begin to understand files, references, previous outputs, and design intent.

For architects, this is the useful direction.

The AI should help carry design logic forward:

- what the project is about
- what has already been tested
- what should stay consistent
- what needs a decision
- what should happen in the next step

That is different from collecting prompts. A prompt collection is useful, but it does not automatically remember the project.

## Nomavek Lab Takeaway

Nomavek Lab is moving toward a board-first AI workflow because small studios need a clearer way to turn AI output into design decisions.

The goal is not to make more random options.

The goal is to make each option easier to review, compare, and continue.

The next useful AI architecture workflow may look less like a prompt box and more like a working board: visual, remembered, and ready for the next decision.

## FAQ

**Q: Is project memory the same as saving chat history?**
A: No. Chat history records the conversation. Project memory should organize the design material: brief, references, options, constraints, and decisions.

**Q: Do small studios really need this?**
A: Yes. Small studios usually have less time to reorganize scattered AI outputs, so a simple board can reduce confusion quickly.

**Q: Does this replace the architect's judgment?**
A: No. It makes judgment easier to apply because the reference, output, and decision question stay visible together.

**Q: Should every AI image become a board?**
A: No. Quick experiments can stay loose. But once an image affects a client direction, competition board, or design decision, it should live inside a clearer project structure.
