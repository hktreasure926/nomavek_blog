# Intake Automation Draft (v1)

Status: draft (no public changes, no messages sent)

Goal: capture a clean project brief, then prepare a small “AI-ready package” for Studio Nomad style direction and rendering.

---

## A) Intake Form (fields)

### Contact
- Name
- Email
- Company (optional)
- Timezone

### Project basics
- Project type (house / apartment / retail / office / hospitality / other)
- Location (city, country)
- Stage (concept / schematic / DD / competition / marketing)

### What you need
- Output type (concept directions / render refinement / style swap / diagram)
- Views needed (list)
- Deadline

### Inputs you can provide
- Base image(s) (upload / link)
- Sketches (upload / link)
- Plans (optional)
- Reference images (3–8 max)

### Constraints (must not change)
- Massing locked? (yes/no)
- Camera locked? (yes/no)
- Materials to keep / avoid

### Taste + intent (simple)
- 3 keywords
- 1 sentence: what the image must communicate

---

## B) Auto-Generated Internal Package (what the system produces)

1) Brief summary (one paragraph)
2) Locked constraints (short list)
3) Reference set (cleaned: remove duplicates)
4) Recommended route (Massing to Render / Style Swap / Touchup / Change Angle / Make Diagram)
5) First proposal: 3 directions (Safe / Stretch A / Stretch B)

---

## C) Human Checkpoint (before any generation)

- Confirm: camera locked vs flexible
- Confirm: “one primary material direction” (yes/no)
- Confirm: any sensitive content or privacy constraints
